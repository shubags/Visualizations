Project Description

The purpose of this assignment was to:

  • Gain basic familiarity with HTML and CSS to create webpages
  • Begin exploring the capabilities of JavaScript to build webpage functionality
  • Learn the basic capabilities of D3 and the process for building D3 objects for visualizations

Functionality was built in phases to successively build on features. The following phases of describe the features added.

Basic functionality (C-Level)

  • Decoded URI Search term
  • Kept only alpha-numeric characters
  • Logged js object with lettercount in console
  • Built styled bars using D3 in an SVG
  • Added text annotation to SVG barsto indicate frequency of letter occurance
  • Ordered bars in alphabetical order (with numbers before letters)
  • Added fixed number of tick marks (5) to Y-axis with absolute frequencies and no decimals

Basic+ functionality (new C-level)

  • Changed bar order to be sorted by frequency (highest to lowest) instead of alphabetical
  • Made graph adjustable to show relative frequencies instead of absolute frequencies
    • Bar data labels change to relative frequency
    • Y-axis labels switch to 2 decimals instead of whole numbers
    • Bar labels switch to 2 decimals instead of whole numbers
  • Made the number of Y-axis ticks dynamic to adjust for the following conditional:
    • If max frequency is less than 10, then number of ticks is the number of max frequency
    • If max frequency is less than 10, then the number of ticks as close to 10 as possible without exceeding where the tick delta is a whole number
    • If the chart type is relative instead of absolute, it sets the tick number at 5
  • Made SVG width and height a function of the window size so that it always fits

Moderate functionality (B-Level)

  • Tool tip feature was added to create pop-up box of bar height on hover-over.
  • A button was added to allow sorting alphabetically or by frequency

Design decisions

The design decisions of this chart can be broken down into two primary categories: aesthetics and data representation. These factors were meant to improve the understandability of the data.

From an data representation perspective, I chose to sort the data by order of frequency (from left to right). This style plot gives users the quickest visual understanding of the most frequently represented letters. It also helps users quickly see how many letters it takes to describe most of the data (e.g. the Pareto principal).

From an aesthetics perspective, I chose a bright blue bar because it is easy to see, and it is the color that is least frequently associated with affected by color blindness. I elected to place the text representing the actual frequency of the letters onto the bars themselves. This is a bit of double encoding, but I wanted users to be able to see what the levels were at in the event that many unique letters and numbers were used. At 26 letters and 10 digits, the bar chart could have 36 elements, making it hard to track the data all of the way to the y-axis. I chose the numbers to be inside of the bars (rather than on top of the bars) because, in the event I added grid lines, I did not want the two elements getting too cluttered. Later on, this data was removed after the tool tip feature was added and users could identify the values by hovering.
